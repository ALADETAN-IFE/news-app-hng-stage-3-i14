# Before you start,
# You should know this.

## Not all screens are designed, but we have a few screens  and assets that can hg

You are tasked with converting an existing responsive website into a mobile app design. This is not a full redesign.

#### Provided Resources
- Colors
- Typography
- Spacing system
- Core UI components

Use these as your source of truth.

---

What You Need to Know
- Not all screens are designed yet.
- Study the original website (mobile responsive view) and the existing designs in this file.
- Complete the remaining screens by adapting existing elements into a proper mobile app experience.

---

How to Approach the Task
- Do not copy the website blindly.
- Reuse existing components from this file.
- Maintain consistency across all screens.
- Adapt layouts to feel natural on mobile.
- Think like a product designer, not just a visual designer.

---

Key Adjustments to Make
1. Navigation
   - Use the provided bottom navigation.
   - Limit to 4–5 primary items.
   - Avoid hamburger-style navigation.

2. Header
   - Make headers bigger and bold.
   - Keep it simple and consistent.
   - Include only essential elements (logo, notifications, auth if needed).
   - Avoid large or crowded headers.

3. Touch & Spacing
   - Ensure buttons and clickable elements are easy to tap.
   - Maintain consistent spacing using the defined system.

4. Layout Flow
   - Design for smooth vertical scrolling.
   - Avoid section-by-section website blocks.
   - Make content feel continuous and natural.

5. Component Consistency
   - Do not create new styles unless necessary.
   - Reuse buttons, cards, and text styles exactly as defined.
   - Avoid slight variations.

6. Interactions
   - Consider tap states and feedback.
   - Think about loading and empty states where applicable.

7. Mobile Behavior
   - Design screens as part of an app, not a browser.
   - Use proper back navigation patterns.
   - Prefer full screens or bottom sheets over web-style popups.

---

Constraints
- Do not introduce new colors or typography styles.
- Stick to the defined spacing system.
- Prioritize clarity and usability over creativity.

---

What We Are Evaluating
- Your ability to stay consistent.
- Your judgment in adapting layouts.
- How well your screens feel like a real mobile app.

---

If anything is unclear, make reasonable assumptions and stay consistent with existing patterns.
