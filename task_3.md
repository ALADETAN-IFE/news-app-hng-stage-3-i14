---

## 📝 Task Description
You are required to build either a **weather forecast app** OR a **news app**. Both options must fetch data from public APIs and deliver a seamless user experience with animations.

Choose **ONE** of the following apps based on your interest:

### ☀️ OPTION A: WEATHER FORECAST APP
Build a weather app with any design sourced freely that provides users with real-time weather information and forecasts. Your app must fetch weather data from public APIs such as: OpenWeatherMap API, Weather API, Tomorrow API

- 🌡️ Current Weather Display: show temperature, weather conditions (sunny, rainy, cloudy, etc.), humidity, wind speed, etc., while displaying appropriate weather icons based on conditions
- 📍 Location-Based Weather: allow users to view weather for their current location and search for weather in any city manually
- 📅 Forecast View: display a multi-day forecast (minimum 3-5 days) and hourly forecast for the current day (optional but recommended)

### 📰 OPTION B: NEWS APP
Build a news app using this [design](https://www.figma.com/design/00VTaCVowX6JSVGUADJyPv/Fresh-reads?node-id=1-1663&t=PloJsXdN5XEuR7wc-1) that collects articles from various sources. Your app must fetch news data from public APIs such as: [NewsAPI](https://newsapi.org/), [GNews](https://gnews.io/), [The Guardian API](https://open-platform.theguardian.com/), [NewsAPIAI](https://newsapi.ai/)

- 🗞️ Article Feed: display a list of articles with headlines, images, descriptions, and publication dates with source information for each article.
- 🏷️ Category Browsing: allow users to browse news by categories (e.g., Technology, Sports, Business, Health) and implement category tabs or a navigation drawer
- 🔍 Article Search: users should be able to search for articles by keywords and display search results in real time.
- 📖 Article Details: open articles in an in-app web view or a details screen with a function to save/bookmark articles for later reading (optional)

---

## 🛠️ Regardless of which option you choose, your app must implement the following:
- 💾 Offline Caching
- ⚠️ Error Handling (no internet connection, location permission denied (weather app only) and API failures/rate limits, invalid search queries, display user-friendly error messages with retry options)
- ⏳ Loading States (show proper loading indicators while fetching data or implement skeleton loaders)

---

## 🎬 ANIMATIONS (COMPULSORY)
Your app must include animations in at least two of the following areas:

- 🔄 Screen Transitions: animated transitions between screens
- 🪄 List/Grid Items: animated entrance for list items as they appear on screen
- 🎛️ UI Components: animated interactions such as weather icon transitions, expand/collapse animations for detailed forecasts, etc.

> **Note:** Animations enhance user experience by providing visual feedback while creating delight and making the app feel responsive and professional.
